## 概述

Hermes 网关支持**飞书（Feishu/Lark）** 等企业 IM，适合在国内团队部署共享助手。

<Callout type="info">
具体字段名与 webhook 格式以你安装的 Hermes 版本及官方 messaging 文档为准。下列为通用配置流程。
</Callout>

## 准备

1. 在[飞书开放平台](https://open.feishu.cn/)创建企业自建应用
2. 获取 **App ID**、**App Secret**
3. 开通机器人能力，配置事件订阅与消息权限
4. 确保运行 gateway 的机器可被飞书回调访问（公网 URL 或内网穿透）

## 配置 Hermes

<TerminalBlock>
{`hermes gateway setup`}
</TerminalBlock>

在向导中选择 **Feishu / 飞书**，按提示填入凭证。也可在 `~/.hermes/config.yaml` 的 `gateway` / `platforms` 段手动添加（结构见官方文档）。

## 验证

1. `hermes gateway status` 显示飞书通道已连接
2. 在飞书中 @ 机器人发送测试消息
3. 确认仅白名单用户可触发（建议先只允许自己）

## 常见问题

| 问题 | 建议 |
|------|------|
| 收不到消息 | 检查事件订阅 URL、应用已发布、权限已审批 |
| 无回复 | `hermes doctor`；确认模型与 API Key 在 gateway 进程环境中可用 |
| 多人群聊无响应 | 检查群聊 @ 规则与平台级工具开关 |

<Callout type="tip">
团队场景可参考实战项目[团队 Telegram 助手](/labs/team-telegram)的思路，将投递渠道改为飞书。
</Callout>
